<objective>
Add speculative time fallback to the no-CAS reschedule path so Bot 7 can attempt a POST even when `getConsularTimes` returns empty (phantom dates). The feature is gated behind a per-bot DB flag (default false) so it deploys disabled and requires explicit user enablement.

Purpose: 12 of 14 post-February reschedule attempts failed with `no_times`. The dates are real cancellations but the times vanish in 1-2 seconds. Historical data shows only 3 times ever seen at Lima: "07:30", "10:00", "10:15". By trying these times speculatively when `available_times` is empty, the bot gets a chance to grab the slot before it disappears. The DB flag ensures this risky behavior (RESEARCH.md Pitfall 2: unknown if failed POSTs count toward Peru's 2-reschedule limit) never fires without explicit user approval.

Output: Modified reschedule-logic.ts with speculative fallback gated by DB flag + comprehensive tests + schema migration.
</objective>
